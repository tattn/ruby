1 # What's Ruby
2 
3 Ruby is the interpreted scripting language for quick and easy object-oriented
4 programming. It has many features to process text files and to do system
5 management tasks (as in Perl). It is simple, straight-forward, and
6 extensible.
7 
8 ## Features of Ruby
9 
10 * Simple Syntax
11 * **Normal** Object-Oriented features(ex. class, method calls)
12 * **Advanced** Object-Oriented features(ex. Mix-in, Singleton-method)
13 * Operator Overloading
14 * Exception Handling
15 * Iterators and Closures
16 * Garbage Collection
17 * Dynamic Loading of Object files(on some architecture)
18 * Highly Portable (works on many Unix-like/POSIX compatible platforms as
19  well as Windows, Mac OS X, BeOS etc.) cf.
20  http://bugs.ruby-lang.org/projects/ruby-trunk/wiki/SupportedPlatforms
21

68 ## How to compile and install
69 
70 This is what you need to do to compile and install Ruby:
71 
72 1. If you want to use Microsoft Visual C++ to compile ruby, read
73  win32/README.win32 instead of this document.
74 
75 2. If `./configure` does not exist or is older than configure.in, run
76  autoconf to (re)generate configure.
77 
78 3. Run `./configure`, which will generate config.h and Makefile.
79 
80  Some C compiler flags may be added by default depending on your
81  environment. Specify `optflags=..` and `warnflags=..` as necessary to
82  override them.
83 
84 4. Edit `defines.h` if you need. Usually this step will not be needed.
85 
86 5. Remove comment mark(`#`) before the module names from `ext/Setup` (or add
87  module names if not present), if you want to link modules statically.
88 
89  If you don't want to compile non static extension modules (probably on
90  architectures which do not allow dynamic loading), remove comment mark
91  from the line "`#option nodynamic`" in `ext/Setup`.
92 
93  Usually this step will not be needed.
94 
95 6. Run `make`.
96 
97 7. Optionally, run '`make check`' to check whether the compiled Ruby
98  interpreter works well. If you see the message "`check succeeded`", your
99  ruby works as it should (hopefully).
100 
101 8. Run '`make install`'
102 
103  This command will create following directories and install files onto
104  them.
105 
106  * `${DESTDIR}${prefix}/bin`
107  * `${DESTDIR}${prefix}/include/ruby-${MAJOR}.${MINOR}.${TEENY}`
108  * `${DESTDIR}${prefix}/include/ruby-${MAJOR}.${MINOR}.${TEENY}/${PLATFORM}`
109  * `${DESTDIR}${prefix}/lib`
110  * `${DESTDIR}${prefix}/lib/ruby`
111  * `${DESTDIR}${prefix}/lib/ruby/${MAJOR}.${MINOR}.${TEENY}`
112  * `${DESTDIR}${prefix}/lib/ruby/${MAJOR}.${MINOR}.${TEENY}/${PLATFORM}`
113  * `${DESTDIR}${prefix}/lib/ruby/site_ruby`
114  * `${DESTDIR}${prefix}/lib/ruby/site_ruby/${MAJOR}.${MINOR}.${TEENY}`
115  * `${DESTDIR}${prefix}/lib/ruby/site_ruby/${MAJOR}.${MINOR}.${TEENY}/${PLATFORM}`
116  * `${DESTDIR}${prefix}/lib/ruby/vendor_ruby`
117  * `${DESTDIR}${prefix}/lib/ruby/vendor_ruby/${MAJOR}.${MINOR}.${TEENY}`
118  * `${DESTDIR}${prefix}/lib/ruby/vendor_ruby/${MAJOR}.${MINOR}.${TEENY}/${PLATFORM}`
119  * `${DESTDIR}${prefix}/lib/ruby/gems/${MAJOR}.${MINOR}.${TEENY}`
120  * `${DESTDIR}${prefix}/share/man/man1`
121  * `${DESTDIR}${prefix}/share/ri/${MAJOR}.${MINOR}.${TEENY}/system`
122 
123 
124  If Ruby's API version is '*x.y.z*', the `${MAJOR}` is '*x*', the
125  `${MINOR}` is '*y*', and the `${TEENY}` is '*z*'.
126 
127  **NOTE**: teeny of the API version may be different from one of Ruby's
128  program version
129 
130  You may have to be a super user to install ruby.
131 
132 
133 If you fail to compile ruby, please send the detailed error report with the
134 error log and machine/OS type, to help others.
135 
136 Some extension libraries may not get compiled because of lack of necessary
137 external libraries and/or headers, then you will need to run '`make distclean-ext`'
138 to remove old configuration after installing them in such case.
